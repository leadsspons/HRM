Hostinger FTP 서브계정 — 만들기 & 삭제하기

개발자에게 내 비밀번호 대신 "임시 FTP 계정"만 주고, 끝나면 지우는 방법

🟢 1단계 · FTP 서브계정 만들기

이 사이트 폴더만 보이는 계정을 새로 만들어 개발자에게 전달합니다
  1. hPanel에 로그인합니다. hpanel.hostinger.com
  2. 위쪽 Websites → 해당 사이트의 Dashboard(관리)로 들어갑니다.
  3. 왼쪽 메뉴에서 Files → FTP Accounts 를 엽니다. Websites› Dashboard› Files› FTP Accounts
  4. "Create a new FTP account" 항목을 채웁니다 (아래 화면 참고).
    hpanel.hostinger.com › FTP Accounts
    Create a new FTP account
    👤 FTP username
    dev_temp
    🔑 Password
    ••••••••••••
    📁 Directory (이 사이트 폴더만)
    /domains/yoursite.com/public_html
    Create
  5. Create 를 누르면 끝. 이 username / password / 서버주소만 개발자에게 줍니다.
✅ 개발자에게 주는 것은 이 3가지뿐입니다: FTP Host (예: ftp.yoursite.com) Username (dev_temp) Password
내 hPanel 로그인 비밀번호는 절대 주지 않습니다.

🔴 2단계 · 작업 끝나면 삭제하기

계정 하나만 지우면 개발자 접근이 즉시 끊깁니다
  1. 같은 화면 Files → FTP Accounts 로 다시 들어갑니다.
  2. 아래에 만들어 둔 계정 목록이 보입니다. 방금 만든 계정을 찾습니다.
    hpanel.hostinger.com › FTP Accounts
    FTP accounts (목록)
    👤 dev_temp  /public_html
    🗑️ Delete
  3. 계정 옆 🗑️ Delete(휴지통) 를 누르고 확인합니다. → 접근 차단 완료 ✅
⚠️ 마무리 보안 2가지 — ① 내 hPanel 비밀번호도 한 번 바꿔주고, ② 프로필 → Security 에서 2FA(2단계 인증) 를 켜두면 끝입니다. 개발자가 봇 토큰이나 service_account.json 같은 열쇠 파일을 봤다면 그것도 새로 발급하세요.
