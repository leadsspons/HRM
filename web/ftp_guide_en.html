Hostinger FTP Sub-account — Create & Delete

Give your developer a temporary FTP account instead of your password — then delete it when done
EN 🇰🇷 한국어

🟢 Step 1 · Create an FTP sub-account

Make an account limited to this site's folder and hand it to the developer
  1. Log in to hPanel. hpanel.hostinger.com
  2. Open Websites → your site's Dashboard.
  3. In the left menu open Files → FTP Accounts. Websites›Dashboard› Files›FTP Accounts
  4. Fill in "Create a new FTP account" (see the screen below).
    hpanel.hostinger.com › FTP Accounts
    Create a new FTP account
    👤 FTP username
    dev_temp
    🔑 Password
    ••••••••••••
    📁 Directory (this site only)
    /domains/yoursite.com/public_html
    Create
  5. Click Create. Give the developer only this username / password / FTP host.
✅ Hand over only these three: FTP Host (e.g. ftp.yoursite.com) Username (dev_temp) Password
Your hPanel login password is never shared.

🔴 Step 2 · Delete it when the work is done

Removing this one account instantly cuts off the developer's access
  1. Go back to Files → FTP Accounts.
  2. You'll see the list of accounts below. Find the one you created.
    hpanel.hostinger.com › FTP Accounts
    FTP accounts (list)
    👤 dev_temp  /public_html
    🗑️ Delete
  3. Click 🗑️ Delete next to it and confirm. → Access revoked ✅
⚠️ Two final security steps — ① change your own hPanel password, ② turn on 2FA under Profile → Security. If the developer saw any key files (bot token, service_account.json), re-issue those too.
