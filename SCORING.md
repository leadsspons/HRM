# 가중 점수 & 인센티브 정책

## 주간 가중 점수 (0~100)
```
score = ( W_완료율 × 완료율%
        + W_준수율 × 완료시간준수율%
        + W_투표  × 투표정규화
        + W_대화  × 대화수정규화 ) / ΣW
```

| 지표 | env | 기본 가중치 | 산식 |
|---|---|---|---|
| 완료율 | `W_COMPLETION` | **35** | 완료 작업 / 배정 작업 × 100 |
| 완료시간준수율 | `W_ONTIME` | **30** | 마감 내 완료 / 완료 작업 × 100 |
| 커뮤니케이션 투표 | `W_VOTE` | 20 | 받은 투표 수를 그 주 최댓값=100으로 정규화 |
| 대화수 | `W_ACTIVITY` | 15 | 메시지 수를 그 주 최댓값=100으로 정규화 |

완료율·완료시간준수율 합 = **65%** → 점수를 지배. 두 지표가 낮으면 상위권 불가.

> **정규화 주의**: 투표·대화수는 "그 주 코호트 내 최댓값 대비 상대값"입니다.
> 따라서 소규모 팀에서 한 명이 메시지/투표를 독식하면 그 항목이 100으로 튈 수 있습니다.
> 완료율이 다소 낮아도 투표·대화에서 앞서면 역전이 가능하니, 이를 막으려면
> `W_VOTE`/`W_ACTIVITY`를 더 낮추거나 `W_COMPLETION`/`W_ONTIME`을 올리세요.


## 답변회신속도 가산점 (⚡ 보너스)
가중 점수(0~100) 위에 **추가**로 더해지는 보너스입니다.
- 측정: 작업 배정(created_at) → 담당자 첫 응답(acknowledged_at)까지의 평균 분.
- 첫 응답은 담당자가 그 작업 그룹에서 보낸 첫 메시지(또는 진행중 전환) 시점.

| 평균 응답 | 보너스 |
|---|---|
| ≤ `RESPONSE_FAST_MIN`(30분) | 만점 `RESPONSE_BONUS_MAX`(+10) |
| 30분 ~ `RESPONSE_SLOW_MIN`(480분) | 선형 감쇠 |
| ≥ 480분(8h) 또는 무응답 | +0 |

```
score = base_score(가중 4지표) + response_bonus
```
완료율·준수율이 같은 두 사람은 **응답이 빠른 쪽이 상위**가 됩니다.
보너스 크기/기준은 `.env`의 `RESPONSE_BONUS_MAX·RESPONSE_FAST_MIN·RESPONSE_SLOW_MIN`로 조정.

## 인센티브 포인트 (주간 → 분기 누적)
- 매주 일요일 23시 집계.
- **완료율 하한 게이트**: `MIN_COMPLETION_FOR_POINTS`(기본 70%) 미만이면 점수가 높아도 포인트 0.
- `MIN_SCORE_FOR_POINTS`(기본 60점) 이상인 멤버만 대상.
- 점수 내림차순 상위 N명에게 `WEEKLY_POINTS`(기본 `10,6,3`) 부여.
- 분기말(1/4/7/10월 1일)에 누적 포인트로 최종 순위 → 인센티브.

## 커스터마이징 예시
```bash
# 완료 중심을 더 강화 (완료·준수 80%)
W_COMPLETION=45  W_ONTIME=35  W_VOTE=12  W_ACTIVITY=8

# 1~5위까지 포인트, 커트라인 70점
WEEKLY_POINTS=12,9,6,4,2   MIN_SCORE_FOR_POINTS=70

# 승인 자동화 폭 넓히기 (신뢰도 0.8 이상이면 자동완료)
AUTO_APPROVE_THRESHOLD=0.8
```
모든 값은 `.env` 수정 후 봇 재시작이면 적용됩니다 (코드 변경 불필요).

## 데이터 출처 (시트 탭)
- 완료율·준수율 ← `Tasks` (status, due_at, completed_at)
- 투표 ← `Votes` (`/vote @동료`)
- 대화수 ← `Activity` (메시지 자동 집계, 매일 flush)
- 주간 결과 ← `Points`, 분기 ← `Quarter`
